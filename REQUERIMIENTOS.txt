LA IDEA DEL PROYECTO (Un poco de narrativa):

El Restaurant "THE SORDIDE" nos contrata para realizar un sistema de reservas de mesas. La idea es poder realizar la reserva a través del mismo
sistema, ademas de que si lo desea, el cliente pueda cargar los datos de los platos que va a querer comer.
Esto se va a hacer mediante un sistema de usuarios, donde una vez dentro tendrá la posibilidad de cargar una reserva.
El administrador del sistema podrá cargar el menu (platos) y ver las ordenes de cada reserva realizada.

Como requerimientos principales tenemos:

El sistema debera permitir gestionar mesas (ABM) (ROL administrador).
    Cada mesa tendra una capacidad de limite.
El sistema debera permitir a los usuarios poder reservar mesas.
El sistema debera verificar la disponibilidad de las mesas y permitir realizar la reserva si hay mesas disponibles.
El sistema debera permitir consultar por mesas disponibles en el momento (opcional).
El sistema debera asignar las mesas de acuerdo a la capacidad solicitada.
El sistema debera contener un sistema de autenticación para usuarios (administrador y cliente).
El sistema debera permitir a los clientes poder registrarse y loguear.
El sistema debera permitir a los usuarios poder cargar su orden dentro de la misma reserva (si lo desean).
El sistema debera permitir a los administradores poder cargar mesas y menues.
El sistema debera permitir a los usuarios poder visualizar el pedido realizado.
El sistema debera permitir a los administradores poder consultar un listado total de las reservas.
