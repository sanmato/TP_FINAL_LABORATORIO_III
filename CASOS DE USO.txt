IDEA: LA IDEA ES HACER UN ABM DE UN SISTEMA QUE MANEJA ADMINISTRACIONES DE CONSORCIOS. UNA ADMINISTRACION PUEDE TENER UNO O MAS CONSORCIOS.
LA ADMINISTRACION TIENE UN USUARIO DONDE SE LE PERMITE LIQUIDAR DEUDAS. PERO NO PUEDE AGREGAR Y ELIMINAR CONSORCIOS.
EL SUPERUSUARIO PODRA GESTIONAR USUARIOS ADEMAS DE AGREGAR Y QUITAR CONSORCIOS EN LAS ADMINISTRACIONES.

EL SISTEMA DEBERA PODER GESTIONAR ADMINISTRACIONES(ABM)
EL SISTEMA DEBERA PODER PERMITIR PODER GESTIONAR CONSORCIOS
EL SISTEMA DEBERA CONTENER MANEJADORES DE ADMINISTRACIONES
EL SISTEMA DEBERA CONTENER USUARIOS
EL SISTEMA DEBERA GESTIONAR LA INFORMACION EN UN JSON DE MANERA PERSISTENTE
EL SISTEMA DEBERA PERMITIR LIQUIDAR DEUDAS
EL SISTEMA DEBERA PERMITIR UN REGISTRO
EL SISTEMA DEBERA PERMITIR UN LOGIN
EL SISTEMA DEBERA PERMITIR GESTIONAR USUARIOS(SUPERUSUARIO)
